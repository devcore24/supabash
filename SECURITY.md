# Security Policy

## Supported Versions

Supabash is in active development (Beta). Security fixes are applied to the latest code only.

| Version / Branch | Supported |
| --- | --- |
| `main` (latest commit) | ✅ |
| Latest release (if any) | ✅ |
| Older releases | ❌ |

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities.

**Preferred**: Use GitHub Security Advisories
- Go to **Security → Advisories → New draft advisory** in this repository.

Include:
- A clear description of the issue and impact
- Steps to reproduce (PoC if possible)
- Affected version/commit
- Any logs or screenshots that help verify the issue

**Response timeline**
- Acknowledgment within **72 hours**
- Status updates at least **weekly** until resolution

If the report is accepted, we’ll coordinate a fix and disclosure timeline with you.
If it’s declined (e.g., not reproducible or out of scope), we’ll explain why.

Thank you for helping keep Supabash secure.
